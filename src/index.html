WYSIWYG HTML Editor for
Human Readable CSS デモ

人間に読みやすいコンポーネント指向のCSS記述法

このページは WYSIWYG HTML Editor for Human Readable CSS のデモページです。ソースコードは GitHub（ https://github.com/storywriter/hrcss ）に公開しています。

1.ドラッグ＆ドロップで、コンポーネントを楽に追加。

ドラッグ＆ドロップページ上部のツールバーから、追加したいコンポーネントを選んで、ドラッグ＆ドロップ。ページ内に、かんたんに追加できます。ためしに、何かを追加してみましょう。

追加した要素の編集は、ダブルクリックで、手軽にできます。

この WYSIWYG HTML Editor for Human Readable CSS は、Human Readable CSS で記述された HTML を解釈して、自在に編集できるようにします。

2.要素を追加できるところに、制約をつくれる。

Human Readable CSS で記述すれば、コンポーネントを追加できるところ、できないところをかんたんに制約できます。また、BEMにおけるブロック、エレメントの層や、リスト、テーブルといった単位で、追加できる場所をコントロールすることができます。

3.コンポーネントの追加はかんたん。Human Readable CSS で記述するだけ。

コンポーネントの追加は、Human Readable CSS で記述するだけでかんたんにできます。ポイントは、ふだん、ウェブ制作者がふれている HTML と CSS の世界だけで、すべてができること。JavaScript や、サーバーサイドの言語を知らない人でも、編集することができます。

さらに良いのは、Human Readable CSS で記述したコンポーネントの一覧表は、そのままウェブページとして表示できます。一覧表を別資料として管理する必要はありません。以下が、コンポーネントの一覧表のサンプルです。

4.読みやすく、学習しやすい。

Human Readable CSS は、人間にとって読みやすいことをコンセプトに設計されています。HTML と CSS の知識があれば、気軽に学習することができます。また、コンポーネント指向で、大規模なウェブサイトでの運用にも耐えられるように設計されています。文法やソースコードは GitHub で公開されています。
